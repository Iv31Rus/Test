git status — показывает текущее состояние файлов в репозитории (какие файлы изменились, удалились, добавились)
git log — показывает историю изменений (это про зафиксированные изменения, то есть коммиты)
git add *name* - Познакомить Git с новым файлом, то есть добавить файл в индекс
git commit -m "comment" - Зафиксировать (закоммитить) изменения 
git push - Синхронизировать изменения с сервером
git branch - Посмотреть, какие ветки есть в проекте и какая сейчас активна
git branch *name* - создание ветки
git checkout *name* - перейти на ветку
git checkout -b *name* - Создать новую ветку
git restore *name* - Вернуть файл в изначальное состояние, отменить изменения
git push --set-upstream origin *name* - Опубликовать коммит и создать новую ветку 
git merge *name* - объединение веток, *name* имя той ветки которую присоединяешь к главной
git config --global user.name "My Name" - добавить глобальную запись в конфиг
git config --list - показать конфиг
git init - создание нового репозитория
git reset - изъять файлы подготавливаемого коммита
git show *hash_commit* - посомтреть коммит по хешу
git remote add origin *url* - подключится к удалённому репозиторию 
git clone *url* - клонировать репозиторий
git pull - скачать изменения с сервера
git stash - добавляет текущие незакоммиченные изменения в стек изменений и сбрасывает рабочую копию до HEAD'а репозитория.
git branch -d *local_branch_name* - удалить локальную ветку
    По мнению Git’а, файл может пребывать в одном из четырёх состояний:
-Неотслеживаемый (untracked).
-зменённый (modified) — файл, в котором есть изменения, но он ещё не добавлен в коммит (не зафиксирован).
-Отслеживаемый (staged) — файл, который добавили в индекс.
-Зафиксированный (committed) — файл уже сохранён в локальной базе, и в нём не было изменений с последнего коммита.
    В связке с состоянием файлов используют три основных секции проекта:
-Рабочая директория (working directory) — это директория, которая содержит в себе то, с чем вы работаете, или то, что вы извлекли из истории проекта в данный момент.
-Область индексирования (staging area) — индекс-файл в каталоге Git, который содержит информацию о том, что попадёт в следующий коммит.
-Каталог Git — место, где Git хранит метаданные и базу объектов вашего проекта.
